Blazor的不同应用

创建ASP.NET Core Web 应用程序时，选择Blazor(ASP.NET Core hosted)或者Blazor(Server-Side in ASP.NET Core)的区别
Blazor(ASP.NET Core hosted)

在此它会自动创建3个项目文件

  1. BlazorCrud.Client - 它具有客户端代码并包含将在浏览器上呈现的页面。(视图)

  2. BlazorCrud.Server - 它具有服务器端代码，例如与DB相关的操作和Web API。（控制器）

  3. BlazorCrud.Shared - 它包含客户端和服务器都可以访问的共享代码。（模型）
    实例应用

Blazor(Server-Side in ASP.NET Core)：服务器端Blazor

在此它会自动创建2个项目文件

  1. ServerSideBlazor.App：这是我们的ASP.NET核心托管项目。该项目包含我们所有的组件逻辑和服务。我们还将在此项目中创建模型和数据访问层。

  2. ServerSideBlazor.Server：这包含我们的服务器端Blazor应用程序。服务器端Blazor应用程序在ASP.NET Core主机应用程序中运行，而不是在浏览器中运行客户端。
